Vim configuration and plugins for C development

!!!IMPORTANT!!! this post is a draft for testing blog publication on GitHub Pages.

Vim one of the most powerful and popular text editor for programing. One of the strongest side of Vim is wide collection of plug-ins for most of use cases. In this post I will tell about my configuration and set of plug-ins for C development.

Requirements and expectations

There are few things that I expect from editor for fast and comfortable work:

  • Fuzzy file search
  • Search for symbols (functions, constants, etc.)
  • Go to definition
  • Auto-completion for structure’s fields (with type information)

More complex helpers like debugger integration, function arguments tool-tip, refactoring, etc. are useful, but I don’t use them for some reasons.

File and symbols search

FZF is powerful plug-in for fuzzy-search for both terminal and Vim. This plug-in supports both search for files with command :Files and for symbols (tags) with command :Tags.

Go to something

CScope is tool which provides ability for “Go to”/search commands on symbols for C code. Your Vim should be compiled with --enable-cscope flag to use this plug-in. To use this plug-in with you code you should generate CScope tags for it using command-line utility. The plug-in supports following commands:

  • :cs find s search for all symbol references
  • :cs find g search for global definition of the symbol
  • :cs find c search for calls to the function
  • :cs find t search for text
  • :cs find e do an egrep search
  • :cs find f search for file name
  • :cs find i search for files which include target file
  • :cs find d search functions that called by target function

For more comfortable use I you mappings similar to cscope_maps for automatic use of name under cursor.

Auto-completion

I use YouComleteMe plug-in for auto-completion for variety of languages include C. The plug-in installation process is not easy since it requires manual compilation of server component. The plug-in has client-server architecture with server written in C++. Fortunately, compilation process is easy and can done in one command. The only problem I face were related to clang compiler (used as a back-end for C/C++), but the case is described in FAQ of the project.

Conclusion

Described settings and plugins are my personal choice and may be not optimal for you. If have any ideas how to improve C/C++ development environment in Vim, you’re welcome to comment your ideas.
