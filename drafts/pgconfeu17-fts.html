PostgreSQL Flexible Full Text Search

This post is textual appendix for PgConf.EU conference talk done by Arthur Zakirov and me. This post contains some additional details about each component of FTS in PostgreSQL, example database with examples code snippets and dataset and scripts used in benchmarks.

Full Text Search problems

There are a lot of systems which require to search on a text data such search bar on a blog, e-market, news site, etc. In all mentioned situations user wants to get some information which contains a word or few which he knows. There are basic solutions like searching for substrings which can be implemented as a regular expression. Since in a lot of languages each word can be presented in different forms and have synonyms, simple pattern matching is not good solution for that type of problems.

Full Text Search is a way to process a text into lexemes, sequence of letters with the meaning represented words. One of the most popular example is representation of a word as a initial form of it. For example word better become good, been translated into be, etc.

An FTS engine in PostgreSQL provides powerful tools for processing text and solve a wide range of text search problems.

Basic FTS components

First of all, lets introduce main terms used in FTS.

  • Token - a sequence of characters which represent a word or other atomic parse element of language. Contains type of token (name) and value.
  • Lexeme - in a PostgreSQL token which were processed by the dictionary and been normalized.
  • Text - a sequence of characters which can be transformed into sequence of tokens.
  • Document - a unit of searching.

It is important to distinguish tokens and lexemes, since tokens represent words after parsing and before dictionary processing, but lexemes represent word after both steps.

Before moving to description of processing and search documents, let’s introduce data types used for FTS.

  • tsvector - contains data of document after processing. Consist of lexemes with optional positions of a document.
  • tsquery - contains data of text search query. Consist of lexemes operands and operators such as AND, OR, NOT and special phrase operator <N>.
